2.2 Guía de implementación de HolaMundoController (rama develop)

A continuación se presenta una guía y más detallada para crear un sencillo controlador "Hola Mundo" en una aplicación Spring Boot. Además de mostrar el código, explicaremos en profundidad lo que realiza cada línea, así como el procedimiento para ejecutar el proyecto.

Código del controlador: HolaMundoController.java

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;
@RestController
@RequestMapping("/saludar")
public class HolaMundoController {
  @GetMapping("/hola") 
	public String  saludar() {
        return "Hola Mundo";
	}
}

Explicación detallada de cada elemento

1. Importaciones:

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

Estas líneas importan las anotaciones que utilizaremos:

  • @RestController: Permite a la clase ser un controlador de Spring que devuelve datos directamente como respuesta a las solicitudes HTTP. En lugar de retornar plantillas HTML, se devuelve, por ejemplo, cadenas de texto o estructuras JSON.
  • @RequestMapping: Sirve para indicar la ruta base (endpoint base) a la cual responderá este controlador.
  • @GetMapping: Especifica que el método que anotemos con esta etiqueta responderá a solicitudes HTTP de tipo GET.

2. Anotación @RestController en la clase:

Esta anotación indica que la clase es un controlador REST. Esto significa que:

  • La clase podrá manejar solicitudes HTTP entrantes.
  • Los métodos dentro de esta clase, al responder, devolverán directamente datos (como cadenas de texto, objetos convertidos a JSON, etc.) sin pasar por una vista.

3. Anotación @RequestMapping("/saludar") en la clase:
Esta anotación establece la ruta base para todas las solicitudes que manejará esta clase. En este caso, cualquier endpoint definido dentro de esta clase se basará en la URL
/saludar. Por ejemplo, si luego definimos un método con @GetMapping("/hola"), la ruta completa para acceder a él será http://localhost:8080/saludar/hola.

4. Definición de la clase HolaMundoController:

public class HolaMundoController {
    ...
}

Aquí simplemente declaramos nuestra clase Java, que funcionará como el controlador. El nombre HolaMundoController es libre, pero procura que sea descriptivo.

5. Anotación @GetMapping("/hola") sobre el método saludar():

@GetMapping("/hola")
public String saludar() {
    return "Hola Mundo";
}
  • @GetMapping("/hola") indica que este método responderá a las solicitudes HTTP GET dirigidas a la URL completa http://localhost:8080/saludar/hola.
  • El método saludar() simplemente devuelve la cadena "Hola Mundo". Al ser un controlador REST, el resultado se enviará directamente al cliente (navegador o herramienta de prueba) que haga la petición.
  • El tipo de retorno String es simple, pero podríamos devolver objetos que Spring Boot convertiría automáticamente a JSON si fuera necesario.

Ejecutar el proyecto Spring Boot

Una vez tengas tu aplicación configurada (por ejemplo, usando Maven o Gradle), puedes ejecutar el proyecto desde la línea de comandos. Si estás usando Maven, emplea el siguiente comando:

./mvnw spring-boot:run

Si todo está configurado correctamente, podrás acceder a la URL http://localhost:8080/saludar/hola desde tu navegador. Al hacerlo, deberías ver el texto "Hola Mundo" mostrado en la pantalla.

2.3 Guía para la Creación e Implementación del “CategoryController”

A continuación se presenta una guía para mostrar datos desde un controlador CategoryController en una aplicación Spring Boot. Se detallará la estructura de archivos recomendada, explicando qué hace cada parte del código.

Estructura de archivos

src
└── main
    ├── java
    │   └── codksv
    │       └── apirfds20242
    │           ├── Business
    │           │   └── Category
    │           │       ├── ResponseObject
    │           │       │   ├── ResponseGetData.java
    │           │       ├── CategoryController.java
    │           ├── ApiRfds20242Application.java
    │           └── CorsConfig.java
    ├── resources
    │   ├── static
    │   ├── templates
    │   └── application.properties

Clase ResponseGetData.java

Esta clase representa el objeto de respuesta que se devolverá en formato JSON al cliente cuando se realice una petición a nuestro endpoint.

public class ResponseGetData {
    public String name;
    public String description;
    public boolean status;
}

Clase CategoryController.java
Esta clase configura un controlador en una aplicación Spring Boot que responde a solicitudes GET en la ruta
/category/getdata, devolviendo detalles en formato JSON usando la clase ResponseGetData

import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

import codksv.apirfds20242.Business.Category.ResponseObject.ResponseGetData;

@RestController
@RequestMapping("/category")
public class CategoryController {

    @GetMapping("/getdata")
    public ResponseEntity<ResponseGetData> getData() {
        // Crear instancia del objeto de respuesta
        ResponseGetData responseGetData = new ResponseGetData();
        
        // Asignar valores a sus campos
        responseGetData.name = "Tecnología";
        responseGetData.description = "Encuentra dispositivos de última generación";
        responseGetData.status = true;

        // Devolver el objeto envuelto en un ResponseEntity con código 200 OK
        return new ResponseEntity<>(responseGetData, HttpStatus.OK);
    }
}

Explicación detallada:

  • @RestController: Indica que esta clase es un controlador REST que responderá con datos (normalmente en JSON) en lugar de vistas HTML.
  • @RequestMapping("/category"): Define la ruta base para este controlador. Esto significa que todas las rutas definidas dentro de esta clase comenzarán con /category.
  • @GetMapping("/getdata"): Indica que el método getData() responderá a solicitudes HTTP GET enviadas a la ruta completa http://localhost:8080/category/getdata.
  • Retorno ResponseEntity<ResponseGetData>:
    • ResponseEntity es una clase que nos permite controlar la respuesta HTTP, incluyendo el código de estado (status code) y los encabezados.
    • ResponseGetData es el tipo de dato a retornar en el cuerpo de la respuesta. Spring Boot se encargará de serializar este objeto a JSON de manera automática.
  • Dentro del método getData() se crea una instancia de ResponseGetData, se rellenan sus campos (name, description, status) y luego se retorna un ResponseEntity con el objeto y el código de estado HttpStatus.OK (200), indicando una respuesta exitosa.

Ejecutar el proyecto Spring Boot

  1. Desde la raíz del proyecto, si estás usando Maven, ejecuta:
./mvnw spring-boot:run

Esto levantará la aplicación en http://localhost:8080.

  1. Abre un navegador web o un cliente para APIs (como Postman o cURL) y realiza una solicitud GET a:
http://localhost:8080/category/getdata

Deberías recibir una respuesta en formato JSON, por ejemplo:

{
  "name": "Tecnología",
  "description": "Encuentra dispositivos de última generación",
  "status": true
}

A continuación se agrega la configuración de CORS y la personalización del archivo application.properties. Estas configuraciones permiten, por ejemplo, acceder a la API desde dominios diferentes a su propio servidor (CORS), así como modificar el nombre de la aplicación y el puerto en el que se ejecutará.

Agregar la clase CorsConfig.java
Esta clase se encarga de configurar CORS (
Cross-Origin Resource Sharing), para permitir que aplicaciones web que se ejecutan en otros orígenes (dominios distintos) puedan acceder a la API. Esto es útil cuando, por ejemplo, se consume la API desde una aplicación Angular o React que se ejecuta en un dominio diferente al de la API.

import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.CorsRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class CorsConfig {
    @Bean
    public WebMvcConfigurer corsConfigurer() {
        return new WebMvcConfigurer() {
            @Override
            public void addCorsMappings(@SuppressWarnings("null") 
            CorsRegistry registry) {
                registry.addMapping("/**")
                    .allowedOrigins("*")
                    .allowedMethods("GET", "POST", "PUT", "PATCH", "DELETE")
                    .allowedHeaders("*");
            }
        };
    }
}

Explicación:

  • @Configuration: Indica que esta clase contiene definiciones de beans y configuraciones para el contexto de Spring.
  • @Bean: Marca el método corsConfigurer() para que Spring lo ejecute al iniciar el contexto, creando un bean de configuración CORS.
  • addCorsMappings: Aquí se definen las reglas CORS, permitiendo a cualquier dominio () acceder a cualquier endpoint ("/**") con diferentes métodos HTTP.

Configurar el archivo application.properties

Este archivo se utiliza para la configuración de parámetros de la aplicación. Puedes ajustar el nombre de la aplicación, el puerto del servidor, entre otras opciones.

spring.application.name=apirfds20242
server.port= 8081

Explicación:

  • spring.application.name: Establece el nombre interno de la aplicación, útil para propósitos de identificación en logs, métricas, etc.
  • server.port: Cambia el puerto por defecto (8080) a otro, en este caso 8081. De esta forma, al iniciar la aplicación, estará disponible en http://localhost:8081.

2.4 Guía para la Creación de un Repositorio en GitHub

A continuación se presenta una guía paso a paso para la creación de un repositorio en GitHub, así como el proceso para subir un proyecto local a dicho repositorio, incluyendo la configuración inicial de Git y la creación de ramas.

  1. Acceder a GitHub: Ingresa a https://github.com con tu cuenta.
  1. Nuevo Repositorio: Haz clic en el botón "New" o en el ícono de "+" y selecciona "New repository".

Crear Repositorio: Haz clic en "Create repository" para finalizar el proceso.

Con ello, tu repositorio estará listo y dispondrás de una URL remota, por ejemplo:

https://github.com/ksvdevs/apirfds20242.git

Comandos para subir el proyecto a gitHub

//Antes de subir el proyecto, es necesario configurar Git con tus datos de usuario 
//para asociar tus commits a tu identidad.
git config --global user.name "ksvdevs"             //Configurar el nombre de usuario:
git config --global user.email "user@gmail.com.pe" //Configurar email

git init   //Inicializa el Repositorio Local
git add .  //Agrega todos los archivos nuevos y modificados.
git commit -m "feat: Initialization"//Crea un punto en la línea de tiempo del repositorio
//con los cambios preparados, etiquetándolos con el mensaje "feat: Initialization"

git branch -M main
git remote add origin https://github.com/ksvdevs/apirfsd20242.git
git push -u origin main //Envía tu rama principal (main) al repositorio remoto

// Comando para crear y cambiar a la rama develop
git checkout -b develop
