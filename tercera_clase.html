2.2 Comunicación de backend con frontend y pruebas con postman.

Consumir un servicio de backend que lista categorías desde un proyecto Angular utilizando

la URL proporcionada (http://localhost:8080/category/getdata)

1. Crear un nuevo proyecto Angular:

ng new appwebds20242
cd appwebds20242

2. Crear un Servicio Angular para la Comunicación con el Backend:

2.1 Genera un servicio llamado category.service:

ng generate service api/category

2.2 En el archivo generado src/app/api/category.service.ts, implementa el método para consumir la API:

import { HttpClient } from '@angular/common/http';
import { Injectable } from '@angular/core';
import { Observable } from 'rxjs';

@Injectable({
	providedIn: 'root'
})
export class CategoryService {

	private apiUrl = 'http://localhost:8081';
	
	constructor(private httpClient: HttpClient) { }
	
	public getData(): Observable<any> {
		return this.httpClient.get(`${this.apiUrl}/category/getdata`);
	}
	
}

Explicación detallada:

2.1 Importaciones:

import { HttpClient } from '@angular/common/http';
import { Injectable } from '@angular/core';
import { Observable } from 'rxjs';
  • HttpClient: Servicio de Angular que permite realizar peticiones HTTP (GET, POST, PUT, DELETE, etc.) a servidores externos.
  • Injectable: Decorador que indica que esta clase puede inyectarse en otros componentes o servicios.
  • Observable: Tipo de dato que emite valores a lo largo del tiempo. En este caso, HttpClient retorna un Observable del que podemos “suscribirnos” para obtener la respuesta asincrónica del servidor.

2.2 Decorador @Injectable({providedIn: 'root'}):

@Injectable({
  providedIn: 'root'
})

Indica que este servicio se registrará en el "root injector" de la aplicación. Esto significa que estará disponible en toda la aplicación sin necesidad de agregarlo en módulos específicos.

2.3 Definición de la clase CategoryService:

export class CategoryService {
  ...
}
  • Es una clase que contiene métodos para interactuar con el backend.
  • Este patrón (crear un servicio) es una buena práctica en Angular para separar la lógica de consumo de datos de los componentes de la interfaz.

2.4 Constructor con inyección de dependencias:

constructor(private httpClient: HttpClient) {}
  • private httpClient: HttpClient inyecta una instancia de HttpClient en el servicio.
  • Gracias a la inyección de dependencias, no es necesario crear ni instanciar HttpClient manualmente; Angular se encarga de ello.

2.5 Variable apiUrl:

private apiUrl = 'http://localhost:8081';

Define la URL base de la API a la que se hará la petición. Tenerla en una variable facilita su modificación sin alterar el resto del código.

2.6 Método getData():

public getData(): Observable<any> {
		return this.httpClient.get(`${this.apiUrl}/category/getdata`);}
  • Realiza una petición HTTP GET a http://localhost:8081/category/getdata.
  • Retorna un Observable<any>, lo que significa que la respuesta del servidor llegará de forma asíncrona.
  • Cuando otro componente se suscriba a este método, recibirá los datos provenientes del servidor (por ejemplo, un objeto JSON) en el callback del subscribe.

2.3. En el archivo src/app/app.config.tsagregar el provideHttpClient

import { ApplicationConfig, provideZoneChangeDetection } from '@angular/core';
import { provideRouter } from '@angular/router';
import { provideHttpClient } from '@angular/common/http';
import { routes } from './app.routes';

export const appConfig: ApplicationConfig = {
  providers: [
    provideZoneChangeDetection({ eventCoalescing: true }), 
    provideRouter(routes),
    provideHttpClient()
  ]
};

Desglose de cada elemento:

3.1 ApplicationConfig:

ApplicationConfig es una interfaz que permite configurar aspectos globales de la aplicación Angular. Al exportar appConfig como ApplicationConfig, estamos definiendo la configuración de inicialización que se aplicará al iniciar la aplicación.

3.2 providers: [ ... ]:

El arreglo providers dentro de appConfig contiene funciones de proveedor que configuran servicios y comportamientos globales. Al utilizar estas funciones, Angular registra servicios y configuraciones para que estén disponibles durante toda la vida de la aplicación.

3.3 provideZoneChangeDetection({ eventCoalescing: true }):

  • Zona y detección de cambios: Angular utiliza "Zone.js" para saber cuándo debe refrescar la vista (correr la detección de cambios) cada vez que ocurre un evento asíncrono (clics, respuestas HTTP, timeouts, etc.).
  • eventCoalescing: true: Esta opción combina (coalesce) múltiples eventos en un solo ciclo de detección de cambios. En otras palabras, si varias acciones ocurren muy rápido unas tras otras, Angular agrupará estas acciones para que la detección de cambios se realice una sola vez, en lugar de hacerlo por cada evento separado. Esto mejora el rendimiento de la aplicación, reduciendo la sobrecarga de actualizaciones de la vista.

3.4 provideRouter(routes):

  • Enrutador de Angular: provideRouter configura el enrutador de la aplicación con las rutas definidas en app.routes.
  • routes: Es un arreglo de objetos ruta que mapean una URL a un componente específico. Gracias a provideRouter(routes), tu aplicación sabrá a qué componente mostrar cuando el usuario navegue a una ruta determinada.
  • Esto hace que puedas cambiar de vista de forma dinámica sin recargar la página, implementando un enrutamiento interno tipo SPA (Single Page Application).

3.5 provideHttpClient():

  • HttpClient: Esta función registra el HttpClient en el nivel raíz de la aplicación. HttpClient es el servicio de Angular que facilita la realización de peticiones HTTP (GET, POST, PUT, DELETE, etc.) a servicios externos o APIs.
  • Al utilizar provideHttpClient(), todos los componentes y servicios de tu aplicación podrán inyectar HttpClient y emplearlo para consumir datos desde el backend, sin necesidad de declararlo en módulos heredados (como ocurría con el antiguo HttpClientModule).

2.4 En el archivo src/app/app.component.ts, utiliza el servicio para obtener las categorías:

import { Component } from '@angular/core';
import { RouterOutlet } from '@angular/router';
import { CategoryService } from './api/category.service';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [RouterOutlet],
  templateUrl: './app.component.html',
  styleUrl: './app.component.css'
})
export class AppComponent {
  category: any = {};

	constructor(
		private categoryService: CategoryService
	) { }
	ngOnInit() {
		this.categoryService.getData().subscribe({
			next: (response: any) => {
				this.category = response;
			},
			error: (error: any) => {
				console.log(error)
			}
		});
	}
}

Explicación detallada:

import { Component } from '@angular/core';
import { RouterOutlet } from '@angular/router';
import { CategoryService } from './api/category.service';
  • import { Component } from '@angular/core';

    Importa el decorador @Component desde el núcleo de Angular. Este decorador se utiliza para indicar que la clase que lo acompaña es un componente.

  • import { RouterOutlet } from '@angular/router';

    Importa RouterOutlet, una directiva de Angular que permite mostrar el contenido asociado a las rutas definidas en el enrutador. Con RouterOutlet, cuando navegas a diferentes rutas, se cargarán dinámicamente los componentes correspondientes en esta área del template.

  • import { CategoryService } from './api/category.service';

    Importa el servicio CategoryService, el cual se encarga de consumir una API externa para obtener datos de categorías. Este servicio se utilizará en el componente para traer la información y mostrarla.

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [RouterOutlet],
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})

@Component({ ... })

Define las opciones del componente.

  • selector: 'app-root': Nombre del elemento HTML que representará este componente en el DOM. Cuando en el index.html aparece <app-root></app-root>, se está insertando este componente.
  • standalone: true: Indica que este componente es independiente (standalone) y no necesita declararse en un módulo. Esta es una característica de las versiones más recientes de Angular.
  • imports: [RouterOutlet]: Permite usar RouterOutlet directamente en este componente. Al ser standalone, necesitamos importar explícitamente cualquier directiva o componente usado en el template.
  • templateUrl: './app.component.html': Ruta al archivo de plantilla HTML del componente, que define la vista.
  • styleUrls: ['./app.component.css']: Ruta al archivo de estilos CSS del componente, que define su presentación.
export class AppComponent {
  category: any = {};
  • export class AppComponent { ... }: Declara la clase del componente. Aquí se define la lógica del mismo.
  • category: any = {};: Declara una propiedad category en la clase y la inicializa como un objeto vacío. Este campo se utilizará para almacenar los datos de la categoría que retornará el servicio. El tipo any indica que puede tener cualquier tipo de dato, lo cual brinda flexibilidad pero menos seguridad de tipos.
 constructor( private categoryService: CategoryService) { }

constructor(private categoryService: CategoryService):

Define un constructor para la clase del componente.

  • private categoryService: CategoryService inyecta una instancia del CategoryService en el componente. Angular se encarga de crear y proveer esta instancia. De esta forma, categoryService se puede usar dentro del componente para llamar al método getData().
  ngOnInit() {
    this.categoryService.getData().subscribe({
      next: (response: any) => {
        this.category = response;
      },
      error: (error: any) => {
        console.log(error);
      }
    });
  }
}
  • ngOnInit():

    Este es un ciclo de vida de los componentes Angular. Se ejecuta una vez que el componente se ha inicializado, es decir, después de que Angular haya configurado todas las propiedades y dependencias. Es un buen lugar para obtener datos externos.

  • this.categoryService.getData().subscribe({...}):
    • this.categoryService.getData() llama al método getData() del servicio. Este método retorna un Observable que emite los datos cuando llegan de la API.
    • .subscribe({...}) se utiliza para "escuchar" los valores emitidos por el Observable. Al suscribirte, defines cómo manejar el resultado:
      • next: (response: any) => { this.category = response; }:Cuando la petición HTTP se completa con éxito y se reciben datos, se ejecuta la función next. La variable response contendrá los datos retornados por la API. Luego, this.category = response; asigna dichos datos a la propiedad category del componente, haciendo que estén disponibles para la vista.
      • error: (error: any) => { console.log(error); }:Si ocurre un error durante la petición (por ejemplo, el servidor no responde o la URL es incorrecta), se llama la función error. Aquí simplemente se registra el error en la consola para diagnóstico.

2.5 En el archivo src/app/app.component.html, 

<h2>Lista de Categorias</h2>
<table>
	<thead>
		<tr>
			<th>Nombre</th>
			<th>Descripcion</th>
			<th>Estado</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>{{category.name}}</td>
			<td>{{category.description}}</td>
			<td>{{category.status}}</td>
		</tr>
	</tbody>
</table>
  • Crea una tabla HTML.
  • La sección <thead> define la cabecera de la tabla.
  • <tr> crea una fila. En este caso, es una fila de cabecera.
  • <th> define una celda de encabezado. Aquí se muestran los nombres de las columnas: "Nombre", "Descripción" y "Estado".
  • <tbody> define el cuerpo de la tabla.
  • Dentro de <tbody> hay una fila (<tr>) con tres celdas (<td>).
  • Cada celda utiliza la sintaxis de interpolación {{ ... }} de Angular, que inserta valores de las propiedades del componente TypeScript en el HTML.
  • {{category.name}} muestra el valor de la propiedad name del objeto category que está definido en el componente.
  • {{category.description}} muestra la propiedad description del objeto category.
  • {{category.status}} muestra la propiedad status del objeto category.

¿De dónde provienen estos datos?

En el app.component.ts, se realiza una suscripción a un servicio (CategoryService) que obtiene datos desde el backend. Cuando la información llega, se almacena en la propiedad category del componente. Este objeto category puede tener campos como name, description y status. Cuando el componente se renderiza, Angular sustituye las expresiones {{ ... }} por los valores actuales de category, mostrando así los datos en la interfaz.

El uso de {{category.name}}, {{category.description}} y {{category.status}} permite mostrar valores dinámicos provenientes del componente de Angular, actualizados en tiempo real cuando el componente recibe la información desde el servicio.

2.6 Ejecutar el Proyecto Angular

  • Ejecuta el proyecto con:
ng serve
  • Abre tu navegador y navega a http://localhost:4200. Deberías ver la lista de categorías obtenida desde el backend.

3. Pruebas con Postman

  1. Verificar el endpoint del backend:
    • Usa Postman para asegurarte de que http://localhost:8081/category/getdata devuelve los datos esperados.
  1. Verificar la comunicación entre Angular y el Backend:
    • Inspecciona la consola del navegador para asegurarte de que no hay errores.
    • Si encuentras un error relacionado con CORS, asegúrate de configurar el backend para permitir solicitudes desde http://localhost:4200.
